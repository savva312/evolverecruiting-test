# Bulgaria agent onboarding playbook (operational v1)

This playbook is written for the UNIC agent manager responsible for Bulgaria. It is designed to onboard a new Bulgarian education agent end-to-end: **pilot → launch → quarterly review**, with clear owners, checklists, and required data formats.

**Inputs to use**
- Target roster: `countries/bulgaria/entities/agents/feeder-candidates.md`
- Agent profiles: `countries/bulgaria/entities/agents/profiles/`

**Non-negotiables (Bulgaria)**
- **Price integrity**: commissions are calculated on **published tuition**; **no student rebates** or “cashback”.
- **GDPR**: agent must collect explicit consent for sharing student data with UNIC (and affiliates) and must be able to evidence consent on request.
- **Single-source truth**: agent uses UNIC-approved assets and the required lead handoff format in this document.
- **Campus clarity**: the agent must not blur **UNIC Nicosia** vs **UNIC Athens** (guardrails below).

---

## Operating model (owners + cadence)

**Core owners**
- **Bulgaria Agent Manager (UNIC)**: owns partner selection, onboarding, weekly cadence, performance management.
- **Admissions Lead — UNIC Nicosia**: owns admissions requirements, file quality, offer turnaround, conversion coaching.
- **Admissions Lead — UNIC Athens**: owns Athens-specific requirements, offer turnaround, conversion coaching.
- **CRM / Admissions Ops (UNIC)**: owns lead intake standards, attribution rules, data hygiene, reporting.
- **Marketing (UNIC)**: owns asset pack, approvals, co-marketing calendar, brand compliance.
- **Finance (UNIC)**: owns commission setup, payment process, invoice requirements.
- **Legal/Compliance (UNIC)**: owns contracting, GDPR/DPA requirements, prohibited claims, termination clauses.

**Working cadence**
- **Week 0–2 (setup)**: due diligence + contract + training + go-live checklist.
- **Weeks 3–12 (pilot)**: weekly 30-minute pipeline review + monthly performance snapshot.
- **Quarterly (QBR)**: scorecard + joint plan (events, targets, improvements).

## Stage 0 — Pre-qualification (1–2 days)

**Goal:** decide whether to invest time into due diligence + contracting.

**Checklist (Agent Manager)**
- Pick candidate from `countries/bulgaria/entities/agents/feeder-candidates.md` and read their profile in `countries/bulgaria/entities/agents/profiles/`.
- Confirm they have **real higher-ed placement** work (not only travel/exchange).
- Confirm they can support **EU applicants (Bulgarian/EU passports)** and also understand non-EU routes when relevant (without over-promising immigration outcomes).
- Confirm they can run **webinars/events** in Bulgarian and can segment messaging by campus (Athens vs Nicosia).
- Confirm they can comply with:
  - **no rebates/cashback**
  - **no guaranteed admission / no guaranteed visa**
  - **GDPR consent + lead sharing**

**Go / No-go rule of thumb**
- **Go** if they can commit to a 90-day pilot, provide weekly reporting, and accept UNIC compliance guardrails.
- **No-go** if they refuse consent evidence, ask for rebates, cannot explain their lead sources, or insist on unapproved scholarship claims.

## Stage 1 — Due diligence (3–5 days)

**Goal:** validate operational readiness and reduce compliance/reputation risk before contracting.

**A) Business verification (Agent Manager)**
- Legal entity name, registration number, VAT (if applicable), office address(es), ownership/beneficial owner statement.
- Evidence of business legitimacy: website, social channels, public reviews, event history, partner logos used (screen for misuse).
- Delivery model: in-house counselors vs sub-agents; list any sub-agents/representatives (names of entities, not individuals).
- Reference checks: 1–2 university partners (or equivalent) willing to confirm performance and payment integrity.

**B) Commercial transparency (Agent Manager + Finance)**
- Student-facing pricing model: confirm what the agent charges students (if anything) and ensure it is transparent and not tied to “rebates”.
- Confirm they will follow UNIC’s **published tuition** and will not advertise unofficial discounts.

**C) Data privacy & security (Compliance + Ops)**
- Confirm GDPR posture:
  - how consent is captured (form language + timestamp + source)
  - how data is stored (CRM, spreadsheets, email)
  - retention policy and deletion requests handling
- Confirm lead transfers will be done via an approved channel (e.g., secure form/portal). No student data via personal email accounts.

**D) Marketing compliance (Marketing)**
- Confirm they will use **UNIC-approved** assets only.
- Confirm they will not use misleading claims or “EU shortcuts” positioning.

**Exit criteria (must be true to proceed)**
- Entity is verifiable and operationally real.
- Agent accepts GDPR consent requirements and the required lead handoff format.
- Agent accepts campus clarity guardrails and prohibited claims list.

---

## Stage 2 — Contracting & setup (5–10 days)

**Goal:** sign a partner agreement that prevents the most common failure modes (price confusion, campus confusion, attribution disputes, data issues).

**Contract components (minimum)**
- **Scope**: Bulgaria recruitment for UNIC Nicosia and/or UNIC Athens (explicitly stated), with program/intake coverage.
- **Commission**: % by program cluster (if different), basis on published tuition, payment timing, clawbacks/chargebacks policy, documentation required.
- **Attribution rules**:
  - required lead submission format + timestamp
  - duplicate handling (e.g., if student already exists in CRM)
  - counselor-of-record rules (no “poaching” between agencies)
- **Marketing & brand**: approvals required; no asset editing without approval; co-marketing calendar.
- **Compliance**:
  - **no student rebates/cashback**
  - no guaranteed admission/visa
  - GDPR/DPA terms and consent evidence requirements
- **Termination**: for compliance breaches (rebates, false claims, misuse of brand, data mishandling), with immediate suspension rights.

**Systems setup (Ops + Finance)**
- Create partner record (partner code / attribution ID).
- Confirm reporting format (weekly pipeline + monthly summary; templates below).
- Confirm invoicing workflow and required finance docs for payouts.

---

## Stage 3 — Training & enablement (5–10 days; overlap Stage 2)

**Goal:** agent counselors can pitch correctly, qualify leads properly, and submit complete files.

**Training agenda (recommended 3 sessions)**
1. **Campus & positioning (60 min)** — Agent Manager + Admissions (Nicosia + Athens)
   - What is UNIC Nicosia vs UNIC Athens (who each campus is right for)
   - Bulgaria-specific objections and answers (value, recognition, relocation, family concerns)
2. **Programs, requirements, and file quality (60–90 min)** — Admissions (Nicosia + Athens)
   - Entry requirements, document checklist, English language requirements, timelines
   - What “application-ready” means; common Bulgaria doc issues to avoid
3. **Lead intake + reporting + compliance (60 min)** — Ops + Compliance + Marketing
   - Required lead handoff fields (below)
   - GDPR consent evidence expectations
   - Prohibited claims and how to use approved messaging/assets

**Certification gate (Agent Manager)**
- At least 2 counselors attend all sessions (or recorded + quiz).
- Agent submits 3 sample leads in the required format (test data) with 100% required fields complete.
- Agent signs acknowledgement of prohibited claims + campus clarity rules.

## Stage 4 — Launch (first 2–4 weeks)

**Goal:** generate first qualified pipeline with clean data and correct campus messaging.

**Go-live checklist (Agent Manager)**
- Confirm a single “go-live date” and the first campaign(s):
  - Bulgaria webinar (split into Athens vs Nicosia segments), and/or
  - in-office info session, and/or
  - fair/event participation with UNIC-approved collateral.
- Confirm the agent’s landing pages/forms include:
  - GDPR opt-in language for sharing data with UNIC
  - campus clarity (student chooses campus interest; not merged)
- Confirm internal UNIC readiness:
  - admissions response SLAs agreed
  - counselor-of-record and duplicate logic operational
  - escalation path for urgent cases (visa timing, deposit deadlines)

**Weekly pilot cadence (Weeks 3–12)**
- 30 minutes: pipeline review (lead volume, quality, stuck cases, next actions).
- Track: lead source, campus split, program demand, document gaps, conversion.

## Stage 5 — 30/60/90-day pilot plan (KPIs + decision rules)

This pilot plan is designed to validate the partner quickly with objective, controllable metrics (quality + pace), not just raw lead volume.

### Day 0–30: “Proof of process”
**Outputs**
- Training completed; sample leads passed.
- 1 joint webinar or 1 in-office event completed.
- Clean lead submissions begin.

**KPIs (minimum)**
- **Data completeness**: ≥90% of submitted leads have all required fields populated.
- **Speed**: agent submits lead to UNIC within 48 hours of first counseling interaction (median).
- **Qualified leads**: at least 15 qualified leads (definition below), with campus split recorded.

### Day 31–60: “Proof of conversion”
**Outputs**
- Regular pipeline cadence.
- First application files (complete documents) submitted.

**KPIs (minimum)**
- **Qualified → Application**: ≥25% of qualified leads reach application submission.
- **Application quality**: ≥80% of applications are “complete on first review” (no major missing docs).
- **Offer velocity**: track offer turnaround; escalate bottlenecks to admissions ops.

### Day 61–90: “Proof of enrollments”
**Outputs**
- At least one intake-specific yield plan (offers → deposits) executed.
- Joint plan for next quarter created.

**KPIs (targets to discuss; calibrate by intake timeline)**
- **Applications → Offers**: ≥60% (varies by program; track by cluster).
- **Offers → Deposit**: set target by intake; treat this as a shared KPI (agent + UNIC).
- **Compliance**: zero incidents of rebates, false claims, or unapproved discounting.

### Decision rules (Day 90)
- **Scale** if KPIs met and pipeline is growing: increase joint events, raise targets, consider tier upgrade.
- **Fix** if volume is good but quality/compliance weak: retrain, tighten SLA, limit programs/intakes temporarily.
- **Stop** if compliance breaches or persistent data/quality failures: suspend immediately and terminate per contract.

**Qualified lead definition (Bulgaria, v1)**
- Student has a defined program cluster, intended intake, and eligibility baseline (education level + English readiness), and has provided GDPR consent to share data with UNIC.

## Stage 6 — Quarterly business review (QBR)

**Inputs (agent submits 48 hours before QBR)**
- Monthly pipeline report (template below) with campus split and outcomes.
- Summary of events/campaigns and lead sources.
- Top 5 objections from Bulgarian families + proposed messaging improvements.

**QBR agenda (60 minutes)**
- Scorecard: lead volume, quality, conversion, compliance.
- What worked / what failed (by channel: fairs, webinars, organic, referrals).
- Next-quarter plan: events calendar, counselor training refresh, program focus.
- Operational fixes: admissions turnaround, document gaps, CRM hygiene.

## Guardrails: UNIC Athens vs UNIC Nicosia (prevent offer confusion)

**Mandatory rule:** every student conversation and every marketing asset must clearly state:
- the **campus** (Athens or Nicosia),
- the **program**, and
- the **intake**.

**What the agent may say (safe framing)**
- “UNIC has two campuses (Nicosia and Athens) with different locations, student experiences, and program availability; we will match you to the best fit.”
- “We will confirm the official entry requirements and tuition for your chosen campus/program before you apply.”

**What the agent must NOT claim**
- “Guaranteed admission” or “guaranteed visa/residence”.
- “Guaranteed scholarship” or any discount not explicitly confirmed in writing by UNIC for that student/intake.
- “EU degree / Schengen access” promises or any immigration outcomes.
- “Same program, same rules, same fees” across the two campuses (unless confirmed for that specific program).

**Campus-routing prompts (use in counseling)**
- If the student prioritizes **living in Greece / Athens lifestyle / mainland EU city** → explore **UNIC Athens** first.
- If the student prioritizes **Medicine/health, value, and Cyprus-based experience** → explore **UNIC Nicosia** first.
- If unsure → present both options clearly as separate offers; do not “bundle”.

## Asset checklist (what the agent gets + what the agent must not do)

**UNIC provides (must be the latest approved versions)**
- Campus comparison one-pager: Athens vs Nicosia (use for Bulgaria counseling).
- Program sheets (per campus) + admissions requirements checklist.
- Tuition/fee schedule + approved scholarship/discount policy (if any).
- Brand guidelines + approved logos + “do not edit” rules.
- Webinar slide deck template + event booth kit (if applicable).
- Lead intake template (fields below) + weekly/monthly reporting templates.
- Escalation directory (roles/inboxes; no personal data in shared docs).

**Agent must not**
- Publish “homemade” tuition tables, altered logos, or unofficial scholarship promises.
- Run paid ads or landing pages using UNIC brand without written approval.
- Submit leads without consent evidence or with missing required fields.

## Required lead handoff format (every lead)

**Rule:** if a lead is missing any required field, it is **not accepted** as an attributed agent lead until corrected.

### Submission format
- Preferred: UNIC-approved lead intake form/portal (if provided).
- If spreadsheet is used: CSV with the columns below, one row per student, UTF-8, no formulas, and a unique `agent_lead_id`.

### Required fields (minimum viable)
| Field | Required | Notes |
| --- | --- | --- |
| `agent_name` | Yes | Agency/company name (not an individual) |
| `agent_lead_id` | Yes | Unique ID created by the agent (stable across resubmits) |
| `consent_captured` | Yes | `yes/no` |
| `consent_timestamp` | Yes | ISO format `YYYY-MM-DDThh:mm:ss±hh:mm` |
| `consent_source` | Yes | e.g., website form, webinar registration, signed consent |
| `student_full_name` | Yes | As in passport/ID if available |
| `student_email` | Yes | One primary email |
| `student_phone` | Yes | Include country code |
| `citizenship` | Yes | e.g., Bulgarian, dual citizenship if applicable |
| `current_city` | Yes | In Bulgaria or current residence |
| `intended_intake` | Yes | e.g., `2026-09` |
| `campus_interest` | Yes | `ATHENS`, `NICOSIA`, or `UNSURE` |
| `program_interest` | Yes | Free text + optionally mapped to a program cluster |
| `education_level` | Yes | e.g., high school, bachelor, master |
| `graduation_year` | Yes | Year |
| `english_level` | Yes | Self-reported + test status |
| `english_test` | No | e.g., IELTS/TOEFL/Duolingo + score/date (if available) |
| `budget_band` | No | Use ranges; avoid collecting unnecessary financial PII |
| `notes` | No | Key constraints, objections, and next steps |

### Recommended additional fields (improves conversion)
- `parent_contact` (only if consented and necessary), `preferred_contact_channel`, `preferred_language`, `event_source`, `school_name`, `gpa_or_grade_avg`, `document_status` (ready/missing), `visa_complexity_flag` (yes/no/unknown).
