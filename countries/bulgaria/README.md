# Bulgaria country workspace

This subtree is the single home for Bulgaria-specific research, playbooks, and data supporting UNIC and UNIC Athens. Keep Bulgaria work isolated from other markets while following the country-first layout.

## Directory layout
- `market/` — outbound mobility, affordability, education system, exams/calendar, and recognition/licensure research for Bulgaria.
- `entities/` — Bulgaria counterpart profiles (schools, agents, regulators, NGOs/SGOs, fairs/events, competitors), including city folders where relevant.
- `go-to-market/` — channel playbooks tailored to Bulgaria (schools, agents, digital, offerholder/yield).
- `program-clusters/` — program-specific positioning, competitor sets, and playbooks with a Bulgaria lens.
- `data/` — structured Bulgaria datasets and dictionaries for entities, marketing, programs, and operations.
- `reports/` — standalone Bulgaria reports and synthesized deliverables.
- `skills/` — Bulgaria-only procedures; global skills stay in `/skills`.

## Working guidelines
- Keep all Bulgaria materials inside `countries/bulgaria/**`; do not cross-mix with other country folders.
- Save reports in `./reports/` and link them from any Bulgaria index pages you add later.
- Use `/executions/` for run logs unless a ticket specifies a different location.
- Follow ticket Allowed/Forbidden paths; ensure any cross-country edits are explicitly permitted.
- For partner mapping and other procedures, start with the global skills (e.g., `/skills/agents-and-partner-ecosystem/`) and then layer the Bulgaria addenda in `./skills/` (see `./skills/README.md`).
