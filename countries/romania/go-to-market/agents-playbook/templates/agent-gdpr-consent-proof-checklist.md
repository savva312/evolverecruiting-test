# Agent GDPR consent proof checklist — Romania

Use this checklist every quarter (or before onboarding) to confirm each Romania-based partner can produce valid consent evidence for sharing leads with **UNIC Nicosia** and **UNIC Athens**. Keep the completed checklist inside the agent’s CRM record (no student PII in the repo).

| Item | Requirement | Evidence link / notes | Status (pass / partial / fail) |
| --- | --- | --- | --- |
| Consent text | Includes UNIC/UNIC Athens name, purpose (program/admissions outreach), withdrawal instructions, DPO/contact email. |  |  |
| Language | Romanian + English versions available; matches what prospects actually see. |  |  |
| Capture method | Screenshot/export of the live form or CRM process showing checkbox or e-sign plus submit button. |  |  |
| Timestamp + source | Log records timestamp, URL/form ID, marketing campaign tag, staff owner (if manually entered). |  |  |
| Data minimization | Lead form collects only approved fields (name, contacts, city, program). No passport/DOB fields. |  |  |
| Storage location | Evidence of secure system (CRM hosted in EU/adequate country, MFA enabled). |  |  |
| Retention timer | Policy or CRM automation deleting/anonymizing leads after 12 months without re-consent. |  |  |
| Retrieval SLA | Demonstrated ability to export consent proof within 5 business days (test request logged). |  |  |
| Sub-agent coverage | Consent proof includes sub-agent attribution where relevant; sub-agent uses the same approved text. |  |  |
| DSAR process | Documented steps + contact person for access/delete requests; response target ≤ 30 days. |  |  |

**Scoring guidance**
- *Pass:* All items satisfied; proceed with normal cadence.
- *Partial:* Gap identified but compensated (e.g., missing Romanian translation). Assign 15-day fix and re-check.
- *Fail:* Consent proof missing, unverifiable, or non-compliant. Freeze referrals until remedied.
