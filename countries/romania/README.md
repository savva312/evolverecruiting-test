# Romania country workspace

Use this subtree for Romania-specific research, execution docs, and data related to UNIC and UNIC Athens. Keep Romania content separated from other markets while following the country-first layout.

## Directory layout
- `market/` — outbound mobility, affordability, education system, exams/calendar, and recognition/licensure research for Romania.
- `entities/` — Romania counterpart profiles (schools, agents, regulators, NGOs/SGOs, fairs/events, competitors), with city folders where useful.
- `go-to-market/` — channel playbooks tailored to Romania (schools, agents, digital, offerholder/yield).
- `program-clusters/` — program-specific positioning, competitor sets, and playbooks with a Romania lens.
- `data/` — structured Romania datasets and dictionaries for entities, marketing, programs, and operations.
- `reports/` — Romania reports and synthesized deliverables.
- `skills/` — Romania-only procedures; global skills stay in `/skills`.

## Working guidelines
- Keep all Romania materials inside `countries/romania/**`; avoid cross-country mixing unless a ticket explicitly allows it.
- Park reports in `./reports/` and use relative links from any Romania index pages you add later.
- Use `/executions/` for run logs unless directed otherwise by a ticket.
- Respect ticket scopes (Allowed/Forbidden paths) before editing outside this subtree.
- Start with global skills (e.g., `/skills/agents-and-partner-ecosystem/`) and then apply the Romania addenda in `./skills/` (see `./skills/README.md`) when drafting playbooks or partner mapping.
