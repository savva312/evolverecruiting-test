# Serbia country workspace

This subtree is the single home for Serbia-specific research, playbooks, data, and skills. Keep all Serbia content here so it stays separated from other markets while aligning with the country-first repo layout.

## Directory layout
- `market/` — outbound mobility, affordability, education system, exams/calendar, and recognition/licensure research for Serbia.
- `entities/` — Serbia-side schools, agents, regulators, NGOs/SGOs, fairs/events, and competitors.
- `go-to-market/` — channel playbooks for schools, agents, digital, and offerholder/yield.
- `program-clusters/` — program-specific clusters with Serbia-tailored competitors and positioning.
- `data/` — structured datasets and dictionaries covering Serbia entities, marketing, programs, and operations.
- `reports/` — Serbia reports and synthesized deliverables.
- `skills/` — Serbia-only procedures; keep global skills in `/skills`.

## Working guidelines
- Keep all Serbia materials inside `countries/serbia/**`; do not mix Serbia work into other country folders.
- Park reports in `./reports/` and link them from any Serbia index pages you add later.
- Use `/executions/` for run logs unless a ticket specifies another location.
- When adding skills, place Serbia-specific procedures in `./skills/**` and keep global skills country-agnostic.
