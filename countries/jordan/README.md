# Jordan country workspace

Use this subtree for Jordan-specific research, execution docs, and data related to UNIC and UNIC Athens. Keep Jordan content separated from other markets while following the country-first layout.

## Directory layout
- `market/` — outbound mobility, affordability, education system, exams/calendar, and recognition/licensure research for Jordan.
- `entities/` — Jordan-side schools, agents, regulators, NGOs/SGOs, fairs/events, and competitors.
- `go-to-market/` — channel playbooks tailored to Jordan (schools, agents, digital, offerholder/yield).
- `program-clusters/` — program-specific positioning, competitor sets, and playbooks with a Jordan lens.
- `data/` — structured Jordan datasets and dictionaries for entities, marketing, programs, and operations.
- `reports/` — Jordan reports and synthesized deliverables.
- `skills/` — Jordan-only procedures; global skills stay in `/skills`.

## Working guidelines
- Keep all Jordan materials inside `countries/jordan/**`; avoid cross-country mixing unless a ticket explicitly allows it.
- Park reports in `./reports/` and use relative links from any Jordan index pages you add later.
- Use `/executions/` for run logs unless directed otherwise by a ticket.
- Start with global skills (e.g., `/skills/`) and then apply the Jordan addenda in `./skills/` when drafting playbooks or partner mapping.
