# Jordan agents and partner ecosystem (addendum)

Use `/skills/agents-and-partner-ecosystem/SKILL.md` as the **primary** procedure for partner mapping. Apply this addendum only for Jordan-specific nuances and paths.

## How to use

- Follow the global workflow for taxonomy, CSV schema, templates, and privacy rules.
- Layer in the Jordan-specific emphases below when executing tickets scoped to `countries/jordan/**`.

## Jordan-specific partner emphases

- **Partner types to prioritize:** generalist agencies with Amman offices plus coverage into Irbid/Zarqa; medicine-focused consultancies serving UK/EU/Cyprus; fair organizers running school/university showcases; counseling hubs embedded in top private/international schools.
- **City tags:** start with `Amman`, `Irbid`, `Zarqa`, `Aqaba`; add secondary cities if the partner actively markets coverage there.
- **Program-fit tags:** default to `medicine`, `cs-data-ai`, `business`; add `engineering`, `hospitality`, or `psychology` only with evidence.
- **Engagement cues:** capture Arabic/English collateral needs, willingness to promote both UNIC Nicosia and UNIC Athens, and preferred webinar/fair formats.
- **Compliance reminders:** confirm GDPR-aligned handling for EU destinations and fee transparency; flag if partners push cash incentives or rebates.

## Quick CSV and profile notes

- Keep `partner_id` slugs consistent with `city-keyword` patterns (e.g., `amman-education-fair`, `irbid-premed-agency`).
- In profiles, note whether partners serve Jordanian students in GCC countries or run hybrid events that include regional audiences.
