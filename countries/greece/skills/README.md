# Greece-specific skills

This folder holds **Greece-only procedures** so guidance stays separate from global skills and other countries’ playbooks. Skills mirror the Bulgaria set for parity (agents, schools, recognition, program clusters, data standards, pricing) and can be expanded as tickets request more detail. For partner mapping, start with the global `/skills/agents-and-partner-ecosystem/` skill and then apply the `greece-agents-and-partner-ecosystem` addendum in this folder.

- Add one `SKILL.md` per Greece procedure and keep metadata accurate (`scope: greece`).
- Do not place Greece content in the global `/skills/` root or other country folders.
- Link relevant Greece skills from the files you update inside `countries/greece/**` so operators can find the right playbook.

Before adding new skills, confirm a ticket allows changes here and avoid duplicating global skills. If the process is country-agnostic, keep it in the global `/skills/` root instead.
