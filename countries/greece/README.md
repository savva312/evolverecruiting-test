# Greece country-first layout

Use this tree for **Greece-specific** content so it stays separated from Bulgaria- and Nigeria-scoped work. It is the single home for Greece research, execution docs, and reports that relate to UNIC / UNIC Athens activities in Greece (market intel, compliance, partner mapping, go-to-market assets).

## Directory map

Create subfolders here as Greece materials are produced. Keep content text-first and link sources.

- `data/` — Greece datasets and dictionaries for entities, programs, marketing, and operations.
- `entities/` — Greece-side counterpart profiles (schools, agents, partners, regulators), including city folders for school profiles.
- `go-to-market/` — Greece-targeted channel playbooks and campaign assets.
- `market/` — Greece market understanding (demand, affordability, recognition/licensure, calendars).
- `program-clusters/` — program-specific positioning, competitor sets, and playbooks with a Greece lens.
- `reports/` — standalone Greece reports (market scans, ops reviews, compliance notes). Keep one file per report and surface key metadata at the top.

If you need a new section not listed above, add it only when scoped by a ticket and keep names consistent with the repo map.

## Contribution rules

- Greece-only material lives under `countries/greece/**`; do not place Greece narratives elsewhere.
- Anchor reports under `reports/` and link them from any Greece index pages you add later.
- Respect ticket **Allowed/Forbidden** paths; if a ticket requires files outside `countries/greece/**`, ensure it explicitly permits that work.
- For partner mapping, use the global `/skills/agents-and-partner-ecosystem/` skill first and then the Greece addendum in `./skills/greece-agents-and-partner-ecosystem/`.

See [`./skills/`](./skills/README.md) for Greece-specific procedures before writing new content.
