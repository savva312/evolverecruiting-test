# Nigeria country workspace

This subtree is the single home for Nigeria-specific research, playbooks, and data. Align new content with the repo’s country-first layout so Nigeria work stays isolated from Bulgaria, Greece, and other markets.

## Directory layout
- `market/` — outbound mobility, affordability, education-system, exams/calendar, and recognition/licensure notes.
- `entities/` — schools, agents, regulators, NGOs/SGOs, fairs/events, and competitors.
- `go-to-market/` — channel playbooks (schools, agents, digital, offerholder/yield).
- `program-clusters/` — program-specific clusters with Nigeria-tailored playbooks.
- `data/` — structured datasets and dictionaries for Nigeria entities, marketing, programs, and operations.
- `reports/` — Nigeria reports and synthesized deliverables.
- `skills/` — Nigeria-only procedures; keep global skills in `/skills`.

## Working guidelines
- Keep all Nigeria materials inside `countries/nigeria/**`; do not use Bulgaria- or Greece-scoped folders.
- Store reports in `./reports/` and use relative links (e.g., `./reports/<filename>.md`).
- Use `/executions/` for run logs unless a ticket specifies otherwise.
- When adding skills, use `./skills/**` for Nigeria-only procedures and keep global skills country-agnostic.
- For partner mapping, use the global `/skills/agents-and-partner-ecosystem/` skill first and then the Nigeria addendum in `./skills/nigeria-agents-and-partner-ecosystem/`.
