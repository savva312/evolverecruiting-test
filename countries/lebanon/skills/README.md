# Lebanon-specific skills

This folder holds **Lebanon-only procedures** so guidance stays separate from global skills and other countries’ playbooks. Shared, cross-country skills remain at the top level of `/skills/`. For partner mapping, start with the global `/skills/agents-and-partner-ecosystem/` skill and then apply the `lebanon-agents-and-partner-ecosystem` addendum in this folder.

- Add one `SKILL.md` per Lebanon procedure and keep metadata accurate (`scope: lebanon`).
- Do not place Lebanon content in the global `/skills/` root or other country folders.
- Link relevant Lebanon skills from the files you update inside `countries/lebanon/**` so operators can find the right playbook.
